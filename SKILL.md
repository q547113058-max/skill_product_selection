# 1688 自动化选品

帮选品人员把"想个关键词→在 1688 一条条翻→比价比销量挑商家→手工整 Excel"这套体力活自动化：
用户用一句话描述想选的产品，你就跑通**关键词→采集→分析→报告**，最后交付一份 Excel 数据表 + 一份 Markdown 分析报告。

报告的核心价值在第四部分"**商家匹配度**"——告诉选品人员每家商家到底专不专做这个品类、靠不靠谱，这正是肉眼翻页最难快速判断、最耗时的部分。

## 关键前置：真实 Chrome 会话（无法自动化的一步）

1688 搜索页有高难度行为滑块，唯一稳过风控的方式是**复用一个已登录、已手动过滑块的真实 Chrome 标签页**（CDP 端口 9222）。登录和过滑块必须人工完成，脚本只负责检测与引导。`run_scraper.py` 会在抓取前自动自检；未就绪时它会打印中文指引，照着转达给用户即可。

## 两套环境路径，别用错

- 所有 python 脚本都用 skill 自带的 venv（`bootstrap.py` 建好后返回它的绝对路径）。
- 采集和分析共用这一个 venv。不要用系统 python，也不要假设用户装了 conda。

约定：下文用 `$PY` 指代 venv 的 python 绝对路径，`$SKILL` 指代本 skill 目录。

## 流程

### 第 0 步：准备环境（首次较慢，之后秒过）

```bash
PY=$(python3 "$SKILL/scripts/bootstrap.py" | tail -1)
```

`bootstrap.py` 会找一个 3.10~3.12 的 Python 建 venv 并装齐依赖（scrapling/playwright/openpyxl/pandas），幂等。首次会装较久（要下载 scrapling 及其依赖），**提前告诉用户在装依赖、稍等**。最后一行输出就是 `$PY`。若它报错说找不到 3.10+ 的 Python，把它的提示转达给用户。

### 第 1 步：需求 → 3-5 个搜索关键词

读懂用户需求，转成 3-5 个适合在 1688 搜索框输入的关键词。思路：
- 拆出**核心品类词**（"船袜""工装袜"），加**同义/上位词**（"袜子""棉袜"）。
- 把需求里的**材质/款式/人群/场景**组合成更精准的长尾词（"纯棉船袜""男士工装袜""加厚毛圈袜"）。
- 覆盖宽窄不同的几个词，既能抓到大盘也能抓到细分，便于后面合并去重看全貌。

需求里若含"销量高/便宜/优质"等**意图**，那是分析阶段的排序/筛选目标，不一定要塞进关键词。

### 第 2 步：让用户勾选关键词（并说明只抓第一屏）

用 AskUserQuestion 把候选关键词列出来让用户**多选**勾选要跑哪些（默认建议跑 2-3 个）。**同时明确告诉用户**：每个搜索词只采集 1688 搜索列表页的**第一屏**商品（约几十个，不翻页）；想覆盖更多就多选几个相关词。

调试或想快出结果时，可对每个词加 `--limit N`（如 `--limit 15`）只抓前 N 个，省时间。

### 第 3 步：检查 / 准备 Chrome 就绪

先自检：

```bash
$PY "$SKILL/scripts/setup_check.py"
```

退出码 0 = 已就绪，直接进下一步。非 0 时**分两种情况，启动 Chrome 这步你来替用户做，别让用户敲命令**：

- **退出码 1（9222 端口不通 = 调试版 Chrome 没起）**：你自己用后台方式启动调试 Chrome：

  ```bash
  bash "$SKILL/vendor/start_chrome_debug.sh" >/tmp/chrome_1688_debug.log 2>&1 &
  ```

  它用一个**专用 profile**（不影响用户日常 Chrome），开 9222 端口并打开 1688 首页。启动后**等几秒让它起来**，然后告诉用户：「已为你打开 1688 的调试浏览器，请在这个新窗口里**登录 1688**，并**手动搜索一次、把滑块拖过去**，弄好后回复我」。登录和滑块**必须用户亲手做**（这步无法自动化）。

- **退出码 2（端口通但没有 1688 标签页 / 未登录）**：Chrome 已经起好了，只差人工。直接提示用户：「请在调试浏览器里登录 1688 并手动搜索过一次滑块，弄好后回复我」。

用户回复弄好后，**再跑一次 `setup_check.py` 确认退出码为 0** 才继续。仍非 0 就把它打印的原因转达用户再等。

### 第 4 步：确定 workspace 并逐个关键词采集

先确定 workspace 放哪——**不要凭空 mkdir，更不要在创建失败后退到用户主目录 `~`**（那是错的）：

1. 先 `pwd` 看当前目录，把它作为默认基准目录。
2. 若用户已指定了存放目录（如"放到 xx 文件夹"），用用户指定的。
3. 在基准目录下按时间戳建子目录，如 `<基准目录>/选品_纯棉船袜_20260629_2100/`。用 `mkdir -p "<workspace>"` 创建。
4. 万一基准目录确实不可写（`mkdir` 报权限错），**停下来问用户希望把结果放在哪个目录**，而不是自己改去主目录。

确定好 `<workspace>` 后，对每个选中的关键词跑：

```bash
$PY "$SKILL/scripts/run_scraper.py" --keyword "纯棉船袜" --workspace "<workspace>" --skip-check
```

第一个词不加 `--skip-check`（让它自检一次）；后续词加 `--skip-check` 避免重复检查。每次成功后它最后一行打印该词的 `1688_<词>.json` 路径，收集起来给下一步。
- 抓取过程中**不要让用户操作那个 Chrome 标签页**。
- 若某词中途被风控拦截，脚本会把被拦商品标记 `[被拦截]` 并继续；分析阶段会自动过滤这些行。整词失败时提示用户在 Chrome 手动重搜过滑块后可单独重跑该词。

### 第 5 步：合并分析

把所有关键词的 json 路径传给分析脚本（`$TS` 用本次时间戳，如 `20260629_2100`）：

```bash
$PY "$SKILL/scripts/analyze.py" \
  --inputs "<ws>/1688_纯棉船袜.json" "<ws>/1688_棉袜.json" \
  --workspace "<workspace>" --demand "<用户原始需求>" --timestamp "$TS"
```

它会合并去重、清洗，产出 `analysis.json`（5 个维度）和 `选品数据_$TS.xlsx`（多工作表）。

### 第 6 步：生成 Markdown 报告（这里你来做语义判断）

读 `<workspace>/analysis.json`，按 `references/report_template.md` 的结构写报告，存为 `<workspace>/选品报告_$TS.md`。

**报告的价值定位（重要）**：本 skill 不是帮用户"从供应商的销量里挑爆品照抄"。最大价值是帮用户**洞察供给**——这个品类在 1688 的**价格分布、款式/卖点、商家质量、商家与用户需求的匹配度**。写报告时聚焦这些，把销量数据只当作**市场需求方向 / 款式验证**的参考信号，明确不要把某家的高销量包装成"要复制的爆款"（同款货是大量商家共同供给的，单店销量更多反映店铺运营而非选品价值）。

报告大部分维度直接用脚本给的数值即可，但**第四部分"商家匹配度"需要你做语义判断**——脚本只给了客观信号（每家的主营品类、类目分布、商品名样本、命中商品、客观商家评分 0~1），故意没下匹配结论，因为关键词匹配判断不准（需求"袜子"而类目叫"船袜/运动袜"会被误判）。你要：
1. 对 `analysis.json` 里 `suppliers` 的每一家，结合用户需求判断**品类匹配**（高/中/低）：看它是不是真的以需求品类为主，搜到的商品是主力还是边缘。注意同义/细分词（船袜/运动袜都算"袜子"）。
2. 把品类匹配（语义）和脚本的"商家评分"（客观）综合成**商家匹配度**，按它排序，优先推荐"高匹配+高评分"的几家，并给出为何匹配/不匹配的一句理由。

需求里如果有脚本没覆盖的特殊角度（如只关心某材质、某价位段、某地区货源），可以基于 `analysis.json` 自行补充一段分析；拿不准要不要加某个维度时，主动问用户。

### 第 7 步：交付

把两份产物的路径告诉用户：
- 分析报告：`<workspace>/选品报告_$TS.md`
- 数据表：`<workspace>/选品数据_$TS.xlsx`

简要口述报告里最关键的结论：主流价格带、款式/卖点分布、商家质量概况、以及和需求最匹配的几家（点名）。

## 资源

- `scripts/bootstrap.py` — 建 venv、装依赖（幂等，返回 venv python 路径）。
- `scripts/setup_check.py` — Chrome 9222 / 1688 标签页就绪自检。
- `scripts/run_scraper.py` — 单关键词采集，产物落到 workspace。
- `scripts/analyze.py` — 合并去重 + 5 维度分析，产出 analysis.json 与 Excel。
- `vendor/` — 已实测的 1688 爬虫（scrape_1688.py / extract.py / start_chrome_debug.sh），勿改。
- `references/report_template.md` — Markdown 报告结构模板。
